# ADR-0003：文件与文件夹导入的持续授权和监听合并

- 状态：已接受
- 日期：2026-07-16
- 修订日期：2026-07-18（菜单选择与 Finder 拖入统一为多文件、多文件夹及混合导入）

## 背景

用户可以通过菜单选择或直接拖入一个或多个视频文件、一个或多个文件夹或两者组合。应用需要在重启后继续访问这些原视频，并在文件移动或重命名后维护其位置。大量文件可能来自同一目录，逐文件创建 FSEvents watcher 会使监听资源随视频数量增长。

## 决策

- 菜单选择和 Finder 拖入生成统一导入请求，共用授权、扫描、判重、入库、刷新和结果反馈流程。
- 每个输入先分类为受支持视频或普通文件夹；符号链接、Package 和不受支持文件不进入导入计划。
- 同一批次先去重；父文件夹覆盖其内部文件和子文件夹，重叠输入只保留覆盖范围最大的来源。
- 每个导入的视频保存文件级 Security-Scoped Bookmark，授权范围保持在用户明确选择或拖入的文件。
- 每个导入的文件夹保存文件夹级 Security-Scoped Bookmark，并递归发现其中的受支持视频。
- 导入 URL 创建 Bookmark 后立即解析为带安全作用域的 URL；扫描、媒体读取、文件监听和后续恢复均使用解析后的 URL，避免把拖放阶段的临时沙盒访问误判为持续授权失败。
- 应用启动和文件事件触发维护时重新解析 Bookmark；解析结果发生变化或 Bookmark 过期时，更新来源 URL、授权记录和监听目录。
- 文件级来源监听其父目录；多个来源映射到同一标准化目录路径时共享一个 FSEvents watcher。
- 任一共享目录事件触发统一防抖维护，维护任务协调全部当前来源并按稳定文件身份更新位置。
- 导入批次逐项目隔离授权、发现和入库失败，其余项目继续处理；视频处理失败与无法读取的输入项目分别统计。

## 影响

- 用户重启应用后仍可访问通过菜单选择或直接拖入的视频，并可在原父目录内移动或重命名后继续使用。
- 用户可以一次导入多个文件夹或文件与文件夹的组合；父子重叠来源不会产生重复授权和重复导入工作。
- 同一目录大量导入视频时，文件事件监听数量由目录数量决定。
- 文件级 Bookmark 不授予整个父目录的持续读取权限；FSEvents 只提供变化通知，原视频访问仍使用各自文件授权。
