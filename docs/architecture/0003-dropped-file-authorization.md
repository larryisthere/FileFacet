# ADR-0003：直接拖入视频的持续授权与监听合并

- 状态：已接受
- 日期：2026-07-16

## 背景

用户可以把一个或多个视频文件直接拖入中部区域。应用需要在重启后继续访问这些原视频，并在文件移动或重命名后维护其位置。大量文件可能来自同一目录，逐文件创建 FSEvents watcher 会使监听资源随视频数量增长。

## 决策

- 每个直接拖入的视频保存文件级 Security-Scoped Bookmark，授权范围保持在用户明确拖入的文件。
- 拖放 URL 创建 Bookmark 后立即解析为带安全作用域的 URL；扫描、媒体读取、文件监听和后续恢复均使用解析后的 URL，避免把拖放阶段的临时沙盒访问误判为持续授权失败。
- 应用启动和文件事件触发维护时重新解析 Bookmark；解析结果发生变化或 Bookmark 过期时，更新来源 URL、授权记录和监听目录。
- 文件级来源监听其父目录；多个来源映射到同一标准化目录路径时共享一个 FSEvents watcher。
- 任一共享目录事件触发统一防抖维护，维护任务协调全部当前来源并按稳定文件身份更新位置。
- 拖入批次逐文件隔离授权、发现和入库失败，失败文件计入结果，其余文件继续处理。

## 影响

- 用户重启应用后仍可访问直接拖入的视频，并可在原父目录内移动或重命名后继续使用。
- 同一目录大量拖入视频时，文件事件监听数量由目录数量决定。
- 文件级 Bookmark 不授予整个父目录的持续读取权限；FSEvents 只提供变化通知，原视频访问仍使用各自文件授权。
