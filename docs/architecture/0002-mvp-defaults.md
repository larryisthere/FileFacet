# ADR-0002：MVP 默认产品与技术决策

- 状态：已接受
- 日期：2026-07-15

## 已确认决策

- 最低支持 macOS 14 Sonoma。
- 支持内置磁盘和外置本地磁盘。
- 排除 NAS、SMB、WebDAV 和其他网络资料库。
- 使用系统 SQLite3，不引入第三方数据库运行时。
- 数据库与缩略图保存在 App Sandbox 中，MVP 暂不增加额外静态加密。
- 设置中提供“启用应用锁”开关，默认关闭。
- 开启应用锁前先完成一次系统身份验证；启用后，Mac 睡眠和系统锁屏后始终重新验证。
- 身份验证开关启用后，应用切换时显示隐私遮罩；开关关闭时保持当前资料库内容可见。
- 闲置锁定按整个 Mac 的无输入时间计算。
- 标签数量包含全部后代标签中的去重视频。
- “最近添加”按首次进入应用索引的时间计算，默认范围 30 天。
- 缩略图缓存默认上限 2GB。
- 视频网格缩略图宽度默认 180 pt；用户调整后保存到本机偏好，并在下次启动时恢复。
- 多个导入来源产生的媒体处理请求合并进入同一后台队列，不因新的来源维护任务取消已有任务。
- 媒体信息或缩略图生成失败后保留失败状态；文件内容发生变化并重新进入待处理状态后再自动尝试，避免每次启动重复处理确定性失败。
- 核心标签操作和主动移出视频支持单步 `Command-Z` 撤销，只保留最近一次成功操作；新的成功操作覆盖上一条撤销记录。
- 标签写操作和视频移出操作按触发顺序串行执行；队列执行期间禁用撤销，失败操作不产生撤销记录。
- 主动移出视频使用单一数据库事务保存撤回快照并删除视频记录；快照包含来源位置、应用标签关系、媒体信息和缩略图引用，原视频保持只读。
- 导入与标签/视频写操作通过任务边界串行协调；导入保留最近一次视频移出撤回，导入状态结束后重新暴露中部撤回入口。
- 视频移出后重新导入同一原视频时，撤回操作复用已重新导入的记录并恢复快照中的标签与来源关系，避免生成重复视频记录。
- 视频移出撤回有效期间保留所需缩略图与来源授权；撤回被新操作覆盖时清理无引用资源，应用启动时再次清理上次进程结束后遗留的无引用缩略图和来源授权。
- Finder 标签在每个视频首次加入索引时单向导入一次；已入库视频后续重扫不刷新其 Finder 标签关系，应用不写回 Finder。
- 用户通过“文件 > 导入视频与文件夹…”、`⇧⌘I` 或 Finder 拖入，一次添加多个视频、多个文件夹或两者组合；所有视频进入统一标签空间。
- 导入按系统文件资源全局判重；文件使用文件级持续访问授权，文件夹使用文件夹级持续访问授权。
- 文件事件监听按实际监听目录合并；同一父目录中的多个文件级来源共享一个 FSEvents watcher。
- 新视频始终由用户手动导入，FSEvents 只静默维护已有视频的位置、内容变化和删除状态。
- 已有视频移动后静默更新位置；来源可访问且确认删除后移除视频记录、标签关系和缩略图。
- 外置卷离线、权限异常和 Bookmark 失效时保留记录。
- 导入发现不跟随符号链接，默认跳过隐藏目录和 macOS Package。
- 默认播放器可能产生自己的最近播放记录，应用不承诺清理外部播放器数据。

## 隐私边界

应用锁启用后用于限制应用入口，并隐藏应用内的视频列表、标签、路径和缩略图。应用锁不加密原始视频，也不为拥有本机账户或磁盘管理权限的人提供完整防护。
