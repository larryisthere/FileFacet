# 手动批量导入视频 Requirements

状态：已确认（2026-07-15）

## 1. 目标

用户可以从任意本地文件夹中手动导入视频，也可以把一个或多个视频文件、一个或多个文件夹或两者混合拖入中部区域，并在同一个 FileFacet 资料库和标签体系中统一管理所有已导入视频。

文件夹用于一次导入操作中的批量选择、递归发现和系统访问授权。应用长期保存视频索引及其标签关系，不向用户提供文件夹资料库列表。

## 2. 用户故事

1. 作为用户，我希望反复选择不同文件夹导入视频，从而统一管理散落在多个位置的素材。
2. 作为用户，我希望系统自动跳过已经导入的系统资源，从而避免重复视频和重复标签关系。
3. 作为用户，我希望导入父目录、子目录或重叠目录时只增加新视频，从而可以放心重复执行导入。
4. 作为用户，我希望已导入视频移动后自动更新位置、删除后自动离开资料库，从而让资料库状态与本地文件保持一致。
5. 作为用户，我希望文件夹中以后出现的视频由我再次手动导入，从而完全控制哪些视频进入 FileFacet。
6. 作为用户，我希望把 Finder 中选中的视频、文件夹或两者组合直接拖入中部区域，从而省去逐次打开文件夹选择器的步骤。

## 3. 功能要求

### 3.1 手动导入

- 用户从 macOS 菜单栏选择“文件 > 导入视频…”发起导入，也可以使用快捷键 `⇧⌘I`。
- 用户可以把一个或多个受支持的视频文件、一个或多个普通文件夹或两者组合直接拖入中部区域发起导入；文件夹递归发现子目录中的视频；拖动经过时保持视频网格原样，只通过系统复制或禁止指针提示能否导入。
- 同一拖入批次应去除重复输入；父文件夹覆盖其内部文件和子文件夹，重叠输入只保留覆盖范围最大的来源。不受支持文件、符号链接和 macOS Package 不进入导入计划。
- 主窗口 Toolbar 和空资料库页面不放置导入按钮；空资料库页面提示菜单路径和快捷键。
- 当用户执行“导入视频”并选择一个本地文件夹时，系统应递归枚举该文件夹及全部子孙文件夹中的受支持视频。
- 当一次导入发现新视频时，系统应将新视频增量加入现有统一资料库，同时保留此前导入的视频和标签。
- 当用户未主动执行导入时，系统不应把来源文件夹中新出现的视频加入资料库。
- 应用启动、回到前台或文件系统发生变化时，系统可以静默维护已有视频的位置和存在状态，但不应自动发现并加入新视频。
- 用户可以重复选择同一文件夹、父目录、子目录或与既有导入范围重叠的目录。
- 每次导入完成后，系统应向用户反馈新增、已存在和失败的视频数量。
- 导入过程和结果应显示在中部标题下方的专属区域；处理中应显示当前阶段，多项目拖入时还应显示批次进度。

### 3.2 全局文件身份与幂等

- 当系统能够读取稳定文件身份时，系统应优先使用卷标识符和文件资源标识符识别同一个系统资源。
- 当稳定文件身份暂时不可用时，系统应使用标准化文件 URL 作为回退身份，并允许后续取得稳定身份时更新记录。
- 当同一个系统资源通过不同导入范围或路径再次被发现时，系统应复用已有视频记录，不新增重复记录，也不重复建立标签关系。
- 当两个文件拥有相同内容但属于两个独立的系统资源时，系统应将其作为两个视频记录。MVP 不执行内容哈希去重。
- 幂等范围覆盖整个 FileFacet 资料库，不受本次选择的文件夹范围影响。

### 3.3 统一资料库

- 所有导入的视频应共享同一个标签树、搜索范围和筛选空间。
- 当用户导入新的文件夹时，系统应保留所有既有视频、应用标签、Finder 标签迁移结果和用户操作历史。
- 当一次导入没有发现某个既有视频时，系统不应据此把该视频标记为无法访问。
- 文件夹不作为用户可见的独立资料库、分类或筛选条件。

### 3.4 已有视频的静默状态维护

- 系统应保存继续访问已导入原视频所需的 Security-Scoped Bookmark：菜单导入和直接拖入的文件夹保存文件夹级授权，直接拖入的视频保存文件级授权。
- 当用户播放视频、在 Finder 中显示视频或读取媒体信息时，系统应访问原始文件，不复制、移动、重命名、修改或删除原视频。
- 当文件事件或启动校验发现已有视频仍存在于任一已授权范围时，系统应按文件资源身份静默更新其路径、文件名和授权来源，并保留应用标签。
- 当系统确认已有视频已经删除时，系统应从资料库移除视频记录、标签关系和缩略图缓存。
- 系统只有在原授权目录可访问、所在卷在线且所有已授权范围内均找不到该文件资源身份时，才能确认删除。
- 当外置卷离线、Bookmark 失效或权限暂时不可用时，系统不应移除视频记录。
- 当文件移动到所有已授权范围以外时，系统无法继续确认其位置；该文件应按离开当前管理范围处理。用户以后手动导入新位置时，系统将其作为新记录加入。
- 状态维护应在低优先级后台任务中运行，不阻塞浏览、筛选、标签编辑、滚动或手动导入。
- 状态维护应默认无界面进度、弹窗或系统通知；成功的路径更新和确认删除直接反映在视频列表中。
- 状态维护只检查已有视频所需的路径、文件身份和访问状态，不读取新视频媒体信息，不生成新缩略图，也不重新同步 Finder 标签。
- 当用户开始手动导入或高频界面操作时，系统应允许状态维护任务降级、暂停或取消。

### 3.5 Finder 标签单向导入

- 当系统首次创建视频记录时，系统应读取一次该文件的 Finder 标签并迁移到应用内部标签。
- 当已有视频在后续手动导入中再次被发现时，系统不应重新读取或同步其 Finder 标签。
- 应用内标签修改不应写回 Finder。

### 3.6 失败隔离与取消

- 当单个文件无法读取或解析时，系统应继续处理本次导入中的其他文件。
- 当文件夹授权失败时，系统应说明失败原因，并保留此前已经导入的视频和标签。
- 当用户取消导入时，系统应停止后续发现和处理；已经完整写入的视频记录应保持有效且不重复。
- 当应用在导入中退出或导入异常中断时，用户再次导入同一范围不应产生重复视频。
- 当拖入批次中的单个文件或文件夹无法授权、读取或写入时，系统应记录该输入项目失败并继续处理其余项目；只有整批项目均无法获得授权时才显示整批失败。
- 结果中的视频失败数量与无法读取的输入项目数量应分开统计；文件夹中没有受支持视频时应显示“未发现可导入的视频”。

## 4. 范围边界

MVP 包含：

- 用户手动选择一个文件夹。
- 用户把一个或多个视频文件、一个或多个文件夹或两者组合直接拖入中部区域。
- 递归发现文件夹及子孙文件夹中的视频。
- 多次导入和全局文件身份去重。
- 统一标签、搜索和筛选。
- 对已导入原视频的持续访问。
- 使用 FSEvents 和启动校验静默维护已有视频的位置与删除状态。
- Finder 标签首次入库单向导入。

MVP 暂不包含：

- FSEvents 自动导入新增视频；FSEvents 只用于维护已有视频状态。
- 自动定时扫描或启动扫描新增视频。
- 用户可见的来源文件夹管理页面。
- 按来源文件夹建立多个资料库。
- 基于文件内容哈希的副本去重。
- Finder 标签双向同步。

## 5. 验收标准

1. 当用户通过“文件 > 导入视频…”或 `⇧⌘I` 选择文件夹 A 时，系统应导入 A 及其全部子孙文件夹中的受支持视频。
2. 当用户随后选择文件夹 B 时，系统应保留 A 中已导入的视频，并只新增 B 中尚未导入的视频。
3. 当用户再次选择 A、A 的子目录或包含同一系统资源的重叠目录时，资料库中的视频数量不应因相同资源增加。
4. 当来源文件夹在导入完成后出现新视频时，系统不应自动加入该视频；当用户再次手动导入该文件夹时，系统应加入该视频。
5. 当两个独立文件的内容相同时，系统应分别显示和管理这两个视频。
6. 当已有视频再次被发现时，系统应保留该视频的应用标签，并跳过 Finder 标签再次同步。
7. 当已有视频在任一已授权范围内移动或重命名时，系统应静默更新其位置并保留应用标签。
8. 当应用重启后，系统应保留所有已导入视频、标签和访问授权，并允许打开可访问的原视频及在 Finder 中显示。
9. 当系统在授权正常且卷在线的条件下确认原文件已删除时，视频应静默离开资料库，其标签关系和缩略图缓存应被清理。
10. 当某个授权范围失效或外置卷离线时，系统应保留相关视频记录，并允许用户继续使用其他可访问视频。
11. 当文件移动到所有已授权范围以外时，该视频应离开当前资料库；用户手动导入新位置后，系统应将其作为新记录加入。
12. 当导入遇到损坏或不支持解析的视频时，系统应保留可识别的基础文件记录，并继续处理其余视频。
13. 当状态维护在后台运行时，用户应能够持续浏览、筛选、滚动、编辑标签和执行手动导入，无需等待维护完成。
14. 当用户把多个视频、多个文件夹或两者组合拖入中部区域时，系统应逐个处理来源并显示阶段和批次进度；其中一个项目失败时，其余可用项目仍应完成导入。
15. 拖动经过中部区域时，视频网格不显示蒙层、虚线框或说明文字；受支持视频使用系统复制指针，不受支持内容使用系统禁止指针。
16. 当应用重启后，直接拖入的视频仍应保留文件级访问授权，直接拖入的文件夹仍应保留文件夹级访问授权；系统应重新解析 Bookmark、更新位置并继续允许打开可访问视频。
17. 当多个直接拖入的视频位于同一父目录时，系统应合并该目录的文件事件监听，避免视频数量直接放大监听资源占用。
18. 当同一批次包含父文件夹、其子文件夹或内部文件时，系统应只处理覆盖范围最大的父文件夹，并保持全局视频判重结果不变。
19. 当拖入的一个或多个文件夹中没有受支持视频时，中部状态区域应显示“未发现可导入的视频”和已检查文件夹数量。
20. 当批次同时存在视频处理失败和无法读取的文件或文件夹时，结果应分别显示两类数量。
