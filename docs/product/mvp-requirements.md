# Mac 原生轻量视频标签管理器

## MVP 产品需求文档

## 产品定位

一款专为 macOS 设计的、本地优先、轻量、高性能的视频标签管理工具。

产品只负责扫描、索引、浏览和标记本地视频，不复制视频文件，不内置播放器，不接入云端，不提供 AI 识别。

核心目标：

> 让用户为指定目录中的视频建立独立于 Finder 的层级标签体系，并快速完成浏览、筛选和批量整理。

---

## 目标用户

需要管理本地视频素材，但不希望使用 Finder 系统标签的 Mac 用户。

典型场景：

* 管理个人拍摄的视频
* 管理项目素材
* 管理私密视频
* 给大量零散视频批量打标签
* 按人物、来源、用途、状态等维度分类
* 使用系统默认播放器播放视频

---

## 产品原则

* 完全原生 macOS 开发
* 极简界面
* 安装包体积小
* 启动速度快
* 不改变原视频文件
* 不依赖网络
* 不绑定任何播放器
* 不写入 Finder 标签，除非未来增加可选同步
* 标签数据只保存在应用内部
* 默认保护用户隐私

---

## MVP 功能范围

### 应用锁

设置中提供“启用应用锁”开关，默认关闭。

关闭时：

* 应用启动后直接进入资料库界面
* Mac 睡眠或锁屏后恢复时不要求身份验证
* App 切换器和窗口失去活动状态时保持资料库内容可见

开启时：

* 开启开关前先完成一次系统身份验证
* 应用下次启动时要求系统身份验证
* Mac 睡眠或锁屏后自动锁定
* 关闭开关后立即解除应用锁

支持：

* Touch ID
* Mac 登录密码回退
* App 重新回到前台时可自动锁定
* 开启应用锁后，Mac 睡眠或锁屏自动锁定
* 可设置闲置自动锁定时间
* 闲置时间按整个 Mac 最近一次键盘、鼠标或滚轮活动计算
* 系统身份验证进行中暂停普通闲置锁定；Mac 睡眠和系统会话锁定仍然优先

推荐选项：

* 立即
* 1 分钟
* 5 分钟
* 15 分钟
* 永不

锁定状态下：

* 隐藏视频列表
* 隐藏标签和文件信息
* 不显示视频缩略图
* App 切换器预览显示空白或模糊占位界面
* 锁定、验证中、验证失败和解锁只切换内容状态，主窗口保持用户当前的尺寸与屏幕位置

这里的应用锁只保护应用入口，不负责加密原始视频文件。应用锁开关默认关闭，用户可以在设置中随时调整。

设置入口与展示：

* 通过应用菜单“设置…”或 `Command-,` 进入设置
* 应用菜单“关于 FileFacet”直接进入主窗口设置的“关于”分类
* 设置显示在主窗口内并独占工作区；原资料库侧栏、视频网格和右侧检查器同时退出，设置分类侧栏与设置内容占满主窗口
* 通过工具栏“资料库”返回原资料库界面，并恢复进入设置前的资料库侧栏和检查器显示状态
* 设置分为“隐私”和“关于”；应用锁与闲置锁定保留在“隐私”中
* “关于”显示当前 `CFBundleShortVersionString` 与 `CFBundleVersion`
* 侧边栏底部不增加设置入口；标签分组的新建按钮维持现有位置

### 手动导入视频

用户从菜单栏选择“文件 > 导入视频与文件夹…”或使用 `⇧⌘I`，一次选择一个或多个受支持视频、普通文件夹或两者组合。应用递归导入所选文件夹及全部子文件夹中的视频，并按所选文件或文件夹保存 Security-Scoped Bookmark，以持续访问已导入文件。

导入可以重复执行。来自多个文件夹的视频进入同一个资料库、标签树、搜索范围和筛选空间。同一个系统文件资源只保留一条视频记录。

资料库行为：

* 不复制视频
* 不移动视频
* 不修改视频
* 不改变目录结构
* 只为用户手动导入的视频建立索引
* 文件夹中以后出现的新视频需要用户再次手动导入

---

### 导入视频发现

用户发起导入后，应用递归发现所选文件夹中的所有子目录。

首期建议支持：

* MP4
* MOV
* M4V
* MKV
* AVI
* WebM

导入结果保存以下信息：

* 文件名
* 相对路径
* 文件扩展名
* 文件大小
* 创建时间
* 修改时间
* 视频时长
* 视频宽高
* 缩略图
* 文件资源标识符
* Finder 标签
* 应用标签

导入过程不能阻塞界面，并提供取消操作与新增、已存在、失败数量反馈。

优先快速显示文件列表，再在后台逐步补全缩略图、时长和分辨率。

---

### Finder 标签迁移

每个视频首次加入资料库索引时，应用读取一次该视频已有的 Finder 标签。

Finder 标签自动转换为应用内部标签。

迁移原则：

* 只读取，不修改 Finder 标签
* 同名 Finder 标签只创建一次
* 已迁移标签不重复创建
* 记录标签来源为 Finder
* Finder 标签迁移后，与应用标签独立存在
* 用户在应用中修改标签，不影响 Finder
* 已入库视频后续再次导入时，不再根据 Finder 当前状态刷新应用内标签关系
* 新视频首次手动导入时执行一次单向导入

Finder 标签本身不支持层级，导入后直接成为顶级标签并平铺展示。顶级已有同名标签时复用该标签并合并视频关系。

示例：

```text
红色
工作
私人
待整理
```

后续用户可以手动移动、重命名或合并这些标签。

---

### 层级标签

用户可以创建多层级标签。

示例：

```text
人物
├── 家人
├── 朋友
└── 同事

用途
├── 工作素材
│   ├── 产品
│   ├── 室内
│   └── 宣传
└── 私人

状态
├── 未整理
├── 已筛选
└── 待删除
```

标签能力：

* 创建标签
* 创建子标签
* 重命名标签
* 删除标签
* 调整标签层级
* 调整标签顺序
* 合并标签
* 为标签设置颜色
* 显示每个标签对应的视频数量

给视频添加子标签后，筛选父标签时默认包含其全部子标签。

---

### 视频标签管理

用户可以对单个或多个视频添加标签。

支持：

* 单选视频
* 多选视频
* 批量添加标签
* 批量删除标签
* 拖动视频到标签
* 拖动标签到视频
* Inspector 中编辑标签
* 快速搜索标签
* 显示最近使用标签

Inspector 中的标签勾选先保存在当前选择的草稿中：

* 点击“应用”后，一次性写入全部新增和移除操作
* 点击“取消”后，恢复到本次编辑前的标签状态
* 草稿存在时切换视频，使用原生确认面板提供“应用”“放弃更改”和“取消”
* 搜索不到同名标签时，可以直接从搜索结果新建标签并选择父标签
* 新标签先以“待新建”状态加入当前草稿并自动勾选
* 标签状态加载完成时，已选中和部分选中的标签优先展示；同组内保持原有顺序
* 勾选、取消勾选和搜索期间保持当前标签顺序，切换视频或重新加载标签状态后重新排序
* 当前草稿中新建的标签置顶展示，多个新标签按最新创建优先排列
* 点击“应用”时，在同一个数据库事务中创建标签并关联当前视频
* 点击“取消”或放弃更改时，同时移除待新建标签，不产生空标签
* 标签分组标题的展开箭头紧跟名称；点击标题主体展开或收起，右侧 `+` 用于新建顶级标签
* 标签分组标题在 `+` 左侧提供排序菜单，可选择“自定义”“按名称”和“按数量 ↓”；排序模式在下次启动时继续保留，使用自动排序时排序图标显示系统强调色
* “按名称”在每个父标签内分别按本地化名称升序排列；“按数量 ↓”按包含自身及全部后代标签的去重视频数从多到少排列，数量相同时按名称升序
* 自动排序只改变展示顺序并保留数据库中的自定义顺序；切回“自定义”恢复原有顺序
* 自定义模式支持调整同层位置和父子层级；自动排序模式继续支持调整父子层级，同层位置由当前排序规则决定
* 点击 `+` 后在标签树中直接进入行内命名；Enter 确认，Escape 取消，失焦确认
* 右键标签选择“重命名…”后，在标签树当前行直接编辑；Enter 或失焦确认，Escape 取消
* 标签名称为空或与同级标签重名时保留编辑状态并提示，当前选中态和标签树展开状态保持不变
* 标题栏只展示当前分类或标签名称及视频数量，不承担标签编辑
* 菜单栏“新建标签…”使用 `Command-Shift-N`，与侧边栏 `+` 执行同一套全局新建逻辑
* 在“标签”和“详情”之间切换时保留草稿
* 当前使用“未打标签”等动态筛选时，应用成功后再刷新结果；不再满足条件的视频随后移出网格
* 视频与标签之间的拖拽操作保持直接生效

批量选择多个视频时：

* 共同拥有的标签显示为已选中
* 仅部分视频拥有的标签显示为中间状态
* 新增标签应用到所有选中视频
* 删除标签从所有选中视频中移除

### 视频浏览

主界面采用 macOS 原生三栏结构：

```text
Sidebar | 视频网格 | Inspector
```

左侧 Sidebar：

* 全部视频
* 未打标签
* 最近添加
* 标签树
* 预置分类与标签条目在右侧显示对应的视频数量

中间视频网格：

* 视频缩略图
* 文件名
* 视频时长
* 可选显示标签
* 支持缩放缩略图尺寸
* 记住用户最近一次设定的缩略图尺寸，并在下次启动时恢复
* 支持单选和多选
* 普通点击建立连续选择锚点；`Shift` 点击选择锚点与目标之间的全部视频
* `Command` 点击增减离散选项；`Command-Shift` 点击将连续区间追加到当前选择
* 选中视频后按空格调用系统 Quick Look；多选时可浏览全部已选视频
* 支持从 Finder 拖入多个视频、多个文件夹或两者组合；文件夹递归发现视频，父文件夹覆盖同批次中的内部文件和子文件夹

右侧 Inspector：

* 缩略图
* 文件名
* 当前完整路径
* 文件大小
* 时长
* 分辨率
* 创建时间
* 修改时间
* 应用标签

打开视频、在 Finder 中显示和复制完整路径通过视频网格右键菜单提供。右键菜单最后一项为“从资料库移除”，支持单选和多选；视频网格获得键盘焦点时也可使用 `Command-Delete`。Finder 标签仅在视频首次导入时迁移为应用内部标签，不在 Inspector 中单独展示。

从资料库移除视频时：

* 操作前显示确认提示，并明确 Mac 上的原视频仍会保留
* 从应用资料库移除视频记录、来源位置和应用标签关系，不移动、修改或删除原视频
* 完成后在中部状态区域提供一次“撤回”，也可使用 `Command-Z`
* 单步撤回应恢复刚才整批移出的视频、来源位置、应用标签和媒体索引；若同一原视频已被手动重新导入，则合并到该记录并恢复原有标签关系，资料库中保持一条视频记录
* 新的成功标签或视频移出操作覆盖上一条撤回记录；失败操作不覆盖
* 导入开始或完成时保留尚未使用的视频移出撤回；导入状态结束后，中部状态区域继续提供该撤回入口
* 缩略图和来源授权在单步撤回仍有效时暂时保留；撤回被后续操作覆盖或应用下次启动后，清理已无视频引用的缩略图、Bookmark、文件访问资源和目录监听
* 已移出视频不会由 FSEvents 自动加回；用户再次手动导入时可以重新加入资料库

Inspector 可以隐藏。

### 搜索与筛选

支持按文件名搜索。

支持按标签筛选。

MVP 筛选规则：

* 点击单个标签：显示该标签及全部子标签中的视频
* 选择多个标签：默认使用 AND 关系
* 提供“未打标签”筛选
* 提供“最近添加”筛选
* 提供清除筛选操作

首期不需要复杂查询语法。

---

### 打开视频

双击视频或点击“打开视频”，使用 macOS 默认关联的播放器打开文件。

不绑定 IINA，不内置播放器，不提供播放器选择器。

技术实现：

```swift
NSWorkspace.shared.open(fileURL)
```

同时支持：

* 在 Finder 中显示
* 复制文件路径

---

### 文件变化处理

应用通过静默维护识别已导入视频的文件变化：

* 删除视频
* 视频重命名
* 视频移动
* 视频内容更新

处理策略：

新增：

* 来源文件夹中新出现的视频不会自动加入索引
* 用户再次手动导入后，新视频才进入资料库

删除：

* 在来源可访问且确认文件已删除后，移除视频记录、标签关系和缩略图
* 外置卷离线、权限异常或 Bookmark 失效时保留记录

重命名或移动：

* 优先通过文件资源标识符重新关联
* 关联成功后保留原标签
* 移动到全部授权范围外时离开当前资料库

## 非功能要求

### 性能

建议目标：

* 冷启动时间低于 1 秒
* 主界面打开后立即显示已有索引
* 扫描和缩略图生成全部后台执行
* 视频网格滚动保持流畅
* 缩略图按需加载
* 支持至少 10,000 个视频
* 标签筛选结果在 200 毫秒内返回
* 不因单个损坏视频中断扫描

### 体积

目标安装包低于 20 MB。

避免引入：

* Electron
* Chromium
* 本地 AI 模型
* FFmpeg 静态打包
* 大型跨平台 UI 框架
* 不必要的第三方 SDK

### 隐私

* 默认无网络请求
* 不采集遥测
* 不上传文件名
* 不上传缩略图
* 不接入 Spotlight
* 不写入系统最近项目
* 缩略图使用 UUID 文件名
* 数据库和缩略图均保存在 App Sandbox 中
* 不在日志中输出完整文件路径和标签内容

## 技术选型

### 开发语言

* Swift

### UI

* 使用 AppKit 构建主窗口和核心视频管理界面
* SwiftUI 仅用于设置、引导、身份验证等轻量辅助界面
* 核心网格使用 `NSCollectionView`
* 标签层级使用 `NSOutlineView`
* 窗口结构使用 `NSSplitViewController`

AppKit 负责：

* 高性能网格
* 键盘操作
* 菜单
* 多选
* 拖拽
* 窗口行为
* App 切换器隐私保护

### 数据库

推荐直接使用系统 SQLite。

不建议首期使用 SwiftData，原因：

* 层级标签需要递归查询
* 视频与标签是多对多关系
* 后续会有大量批量更新
* 文件扫描会涉及事务和 Upsert
* SQLite 更容易控制性能和迁移

如需提高开发效率，可使用 GRDB，但需要评估其对安装包体积的影响。

### 媒体解析

* AVFoundation

用于：

* 视频时长
* 分辨率
* 视频轨道信息
* 缩略图生成

无法被 AVFoundation 解析的视频仍然保留文件记录，只显示基础文件信息。

### 文件权限

* Security-Scoped Bookmark

用于保存对所有已导入来源文件夹的长期访问权限。

### 文件监听

* FSEvents

用于静默维护已有视频的位置、内容变化和删除状态。FSEvents 事件不会创建新视频记录。

### Finder 标签读取

读取文件扩展属性：

```text
com.apple.metadata:_kMDItemUserTags
```

首期只读取，不写入。

### 身份验证

* LocalAuthentication

优先使用：

```swift
deviceOwnerAuthenticationWithBiometrics
```

失败后回退：

```swift
deviceOwnerAuthentication
```

不自行保存独立密码。

### 打开文件

* NSWorkspace

用于：

* 使用默认应用打开视频
* 在 Finder 中显示文件

## 已确认的 MVP 默认决策

* 最低支持 macOS 14 Sonoma
* 支持内置磁盘和外置本地磁盘
* 不支持 NAS、SMB、WebDAV 和其他网络资料库
* 数据库与缩略图保存在 App Sandbox 中，MVP 不增加额外静态加密
* 身份验证开关默认关闭；开启时先完成一次系统验证
* 启用应用锁后，Mac 睡眠和系统锁屏后始终重新验证
* 闲置锁定按整个 Mac 的无输入时间计算
* 闲置锁定默认选择“永不”；“立即”在应用失去活动状态时锁定
* 标签数量包含全部后代标签中的去重视频
* “最近添加”按首次进入应用索引的时间计算，默认 30 天
* 缩略图缓存默认上限 2GB
* 核心标签操作和主动移出视频支持单步 `Command-Z` 撤销，只保留最近一次成功操作
* 新的标签操作或视频移出操作成功后覆盖上一条撤销记录；失败操作不产生撤销记录
* Finder 标签导入和后台维护状态不进入用户标签操作的撤销范围
* 导入发现不跟随符号链接，默认跳过隐藏目录和 macOS Package
* 菜单选择与 Finder 拖入共用授权、扫描、判重、入库和结果反馈管线
* 通过菜单选择或直接拖入的文件保存文件级授权，文件夹保存文件夹级授权；视频失败与无法读取的输入项目分别统计
* 默认播放器可能产生自己的最近播放记录

应用锁用于隐藏应用内的视频列表、标签、路径和缩略图。它不加密原始视频，也不为拥有本机账户或磁盘管理权限的人提供完整防护。

## MVP 完整流程

```text
启动应用
→ 进入统一视频资料库
→ 从“文件 > 导入视频与文件夹…”选择视频、文件夹及其组合
→ 或从 Finder 拖入同类内容
→ 手动导入视频
→ 读取并迁移 Finder 标签
→ 展示视频缩略图
→ 创建层级标签
→ 单个或批量打标签
→ 按标签筛选
→ 双击使用系统默认播放器打开
```

## 验收标准

用户首次启动后，可以在不阅读教程的情况下完成：

* 在设置中按需开启身份验证
* 从不同文件夹重复导入视频
* 从 Finder 一次拖入多个视频、多个文件夹或两者组合
* 看到目录中的视频
* 看到已有 Finder 标签被导入
* 创建一个父标签和子标签
* 给单个视频添加标签
* 给多个视频批量添加标签
* 点击标签筛选视频
* 双击视频使用默认播放器打开
* 在 Finder 中定位视频
* 再次启动应用后保留视频、来源授权和标签数据

满足以上流程，即可视为 MVP 完成。

手动导入、Schema 7、静默维护和主界面交互已经落地；正式窗口行为以用户验收结果为准。
